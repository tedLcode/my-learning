## 📚 What I Studied Today
- Topic: Enterprise Infrastructure Request Process
  - What I understood: Raising infrastructure requests is not a single action — each component (workspace, AD group, service account) is provisioned by a different team and has its own approval queue, so sequencing requests correctly reduces total wait time
  - What I understood: Starting with workspace and AD group requests in parallel (before service accounts) makes sense because service account creation depends on the AD group existing first

- Topic: Environment Segregation in BI Setup
  - What I understood: UAT and PROD are handled as separate provisioning tracks — this ensures that UAT can be validated end-to-end before any PROD components go live, which is the same controlled-rollout principle used in application deployments
  - What I understood: Separate AD groups per environment means permissions can be managed and audited independently — if a PROD permission issue occurs, it doesn't affect UAT governance and vice versa

- Topic: Correct Infrastructure Sequencing
  - What I understood: The dependency order for Power BI enterprise setup is: Workspace → AD Group → Access Configuration → Service Account → Repo — each layer enables the next, so raising requests out of order creates blockers downstream
