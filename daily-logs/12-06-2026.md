## 📚 What I Studied Today
- Topic: Idempotent SQL with IF EXISTS
  - What I understood: Wrapping ALTER statements in `IF EXISTS` guards is essential for scripts that may be run across multiple environments at different points in time — it prevents failures when objects have already been renamed
  - What I understood: This pattern is especially important in DEV/UAT where the script may be rerun during testing without rolling back the database first

- Topic: Deployment Order for DDL/DML Changes
  - What I understood: The correct sequence is ALTER (for existing environments) → CREATE (for fresh environments) → INS (data seeding) — mixing the order causes dependency failures because DML relies on the renamed objects existing first
  - What I understood: Run-order files (`RUN_ORDER_DEV.txt`, `RUN_ORDER_REL.txt`) are how the deployment pipeline knows what to execute and in what sequence — they need to reflect any new scripts added

- Topic: DML and DDL Consistency
  - What I understood: If the DDL renames a table but the INS script still references the old name, the DML fails silently or errors at deployment — always validate all three layers (DDL, ALTER, DML) together before packaging
