## 📚 What I Studied Today
- Topic: Power BI Infrastructure Setup in an Enterprise Environment
  - What I understood: Power BI in an enterprise context is not just "make a dashboard" — it requires proper infrastructure: a dedicated workspace for access control, an AD group to manage permissions at a team level, a service account for automated refresh/deployment, and optionally a Git repo for version-controlling reports and datasets
  - What I understood: Starting with workspace and AD group setup is the right order — permissions and access control need to be in place before any reports or datasets are published, otherwise access management becomes retroactive and messy

- Topic: Active Directory Groups for Power BI Access Control
  - What I understood: An AD group for Power BI centralizes permission management — instead of assigning workspace access to individuals, you add people to the AD group and the group controls access; this scales cleanly as the team grows or changes
  - What I understood: Naming conventions for AD groups matter in enterprise environments — they need to be consistent with existing group naming patterns so IT and security can manage them correctly

- Topic: Git Integration for Power BI
  - What I understood: Power BI has Git integration support (via `.pbip` format and Azure DevOps / GitHub) that allows version control of reports, but it's only needed if the team wants to track report changes over time — the "if needed" qualifier on this subtask reflects that it's optional based on team workflow
